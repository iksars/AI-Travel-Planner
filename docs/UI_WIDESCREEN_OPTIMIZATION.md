# 旅行计划详情页 - 宽屏优化设计说明

## 🎨 设计理念

针对宽屏用户优化了旅行计划详情页的布局，充分利用横向空间，提升信息密度和视觉体验。

## 📐 布局结构

### 1. Hero 区域优化

**之前**：
- 纵向堆叠标题和状态标签
- 下方独立的 4 个信息卡片（占满宽度）

**现在**：
- 左右分布式布局
- 左侧：标题 + 状态标签
- 右侧：4 个信息卡片内嵌在 Hero 区域
- 使用毛玻璃效果（backdrop-filter）
- 响应式网格布局（4列 → 2列 → 1列）

**优势**：
- 充分利用宽屏横向空间
- 减少垂直滚动距离
- 视觉层次更清晰
- 更现代的玻璃态设计

### 2. 主内容区域 - 左右分栏

**新布局**：
```
┌─────────────────┬─────────────────┐
│                 │                 │
│   左侧 (50%)    │   右侧 (50%)    │
│                 │                 │
│  - 地图可视化   │  - 每日行程     │
│  - 预算分析     │    (时间线)     │
│                 │                 │
└─────────────────┴─────────────────┘
```

**左侧内容**：
1. **地图可视化**
   - 更大的地图显示区域（600px 高）
   - 粘性定位（sticky），滚动时保持在视口
   - 完整的地图控制和筛选功能

2. **预算分析**
   - 预算概览卡片
   - 图表可视化
   - 费用记录列表

**右侧内容**：
- **每日行程时间线**
  - 保持原有的时间线设计
  - 可独立滚动
  - 详细的活动、住宿、餐饮信息

### 3. 粘性定位（Sticky Positioning）

地图区域使用粘性定位：
```css
.map-section {
  position: sticky;
  top: 88px; /* 导航栏高度 + 间距 */
}
```

**效果**：
- 用户滚动查看右侧行程时，地图保持可见
- 方便同时查看地图和行程细节
- 提升用户体验

## 🎯 响应式设计

### 宽屏（> 1400px）
- 左右分栏布局，各占 50%
- Hero 区域信息卡片 4 列显示
- 最大容器宽度 1600px

### 中等屏幕（1024px - 1400px）
- 改为单列布局（上下堆叠）
- Hero 信息卡片改为 2 列
- 地图取消粘性定位

### 平板（768px - 1024px）
- 单列布局
- Hero 区域改为纵向排列
- 信息卡片 2 列

### 手机（< 768px）
- 完全单列布局
- 信息卡片 1 列
- 简化间距和字体大小

## 🎨 视觉改进

### 1. Hero 区域
- **玻璃态设计**：使用 `backdrop-filter: blur(10px)`
- **半透明背景**：`rgba(255, 255, 255, 0.15)`
- **柔和边框**：`border: 1px solid rgba(255, 255, 255, 0.2)`
- **hover 效果**：悬停时背景变亮并向上移动

### 2. 信息卡片
- 内嵌在 Hero 区域，与背景融为一体
- 图标使用圆角背景突出显示
- 文字层次分明（标签小写 + 数值大写）

### 3. 容器宽度
- 从 1200px 增加到 1600px
- 更好地利用宽屏显示器

```css
.map-section {

```

## 🎯 设计原则

1. **移动优先，渐进增强**
   - 从手机布局开始设计
   - 逐步增加宽屏特性

2. **内容优先**
   - 重要信息放在显眼位置
   - 次要信息适当隐藏或折叠

3. **视觉一致性**
   - 统一的圆角、阴影、间距
   - 协调的配色方案

4. **性能优化**
   - 使用 CSS Grid 而非 Flexbox（更高效）
   - 最小化重绘和重排
